# EDP员工数据平台 PRD（MVP版）

## 一、产品背景

在企业数字化过程中，员工数据通常分散在多个系统，例如：

- HR系统（HRIS）

- 招聘系统

- 绩效系统

- 考勤系统

- OA系统

这些系统之间存在以下问题：

|问题|描述|
|-|-|
|数据分散|员工数据存在多个系统|
|数据口径不一致|不同系统员工数据不一致|
|数据分析困难|HR需要手动整理数据|
|数据安全风险|权限控制不统一|

因此需要建设 **EDP员工数据平台**，统一员工数据，实现员工数据治理、分析与服务能力。

---

# 二、产品目标

EDP平台核心目标：

1️⃣ 建立 **员工主数据体系（One-ID）**
2️⃣ 整合 **多HR系统员工数据**
3️⃣ 提供 **员工数据服务API**
4️⃣ 提供 **员工数据分析能力**
5️⃣ 引入 **AI员工数据分析助手**

---

# 三、产品定位

EDP平台定位为：

> **企业员工数据的统一数据平台与数据服务层**

系统架构：

# EDP核心数据模型图（最重要）

员工数据平台核心模型：

```Plain Text
员工（Employee）
│
├─ EmployeeID
├─ Name
├─ Gender
├─ Country
├─ HireDate
├─ Status

        │

组织（Organization）
├─ OrgID
├─ OrgName
├─ ParentOrg

        │

岗位（Position）
├─ PositionID
├─ PositionName
├─ JobLevel

        │

绩效（Performance）
├─ EmployeeID
├─ ReviewPeriod
├─ Score

        │

考勤（Attendance）
├─ EmployeeID
├─ WorkDate
├─ AttendanceStatus
```


关系结构：

```Plain Text
组织
  │
  ├── 员工
        │
        ├── 岗位
        ├── 绩效
        ├── 考勤
```


你可以解释一句：

> 员工是核心实体，所有HR数据都围绕员工进行关联。
